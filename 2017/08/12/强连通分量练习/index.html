强连通分量练习

发表于 2017-08-12 | 分类于 acm ， 图论 ， 强连通分量

写完了模板就该来写写题目了，不然光写模板多没有意思。一起来看看我们曾经踩过的强连通分量的坑。

强连通分量一般都伴随着缩点的操作，要你只求强连通分量是没有难度的。当然人家也不可能会告诉你这个题目用强连通分量来解
最典型的入门还是hdu1269题了，只要求出强连通分量的数目然后进行判断就好了；

hdu2767（强连通分量+缩点）

题目很长的一段英文，但是就是一个意思，要你求至少要给一个有向图加多少条边才能使他变成一个强连通图。
如果是第一次做这种题，肯定懵逼了，没错，我也是的！但是做多了以后发现真的好简单啊！它的套路就是把一个有向图的所有强连通分量先求出来，然后再把强连通分量缩点，求出缩点后的图的入度为0的点数和出度为0的点数，谁大就是答案（为什么呢？因为我是老大啊！）
我觉得应该这样理解，要把出度为0和入度为0的点都连接起来就可以变成一个强连通图了，那就是看谁的点多了，因为n个点组成一个强连通图至少需要n条边（一个环）。然后就会发现这个缩点求入度和出度的过程也是非常的精妙。另外还要注意的是当给出的有向图就是强连通分量时，那就不用加边，结果应该输出0，看代码就知道了

类似的题还有poj1236，poj2186

hdu1827（强连通分量+缩点+权值更新）

听说lcy帮大家预定了新马泰7日游，Wiskey真是高兴的夜不能寐啊，他想着得快点把这消息告诉大家，虽然他手上有所有人的联系方式，但是一个一个联系过去实在太耗时间和电话费了。他知道其他人也有一些别人的联系方式，这样他可以通知其他人，再让其他人帮忙通知一下别人。你能帮Wiskey计算出至少要通知多少人，至少得花多少电话费就能让所有人都被通知到吗？

Input
多组测试数组，以EOF结束。第一行两个整数N和M（1<=N<=1000, 1<=M<=2000），表示人数和联系对数。接下一行有N个整数，表示Wiskey联系第i个人的电话费用。接着有M行，每行有两个整数X，Y，表示X能联系到Y，但是不表示Y也能联系X。

Output
输出最小联系人数和最小花费。每个CASE输出答案一行。

hdu3639 （强连通分量+缩点+逆向思维）

这题的意思是：幼儿园的小朋友要玩老鹰抓小鸡，但是很多小朋友的想要当老鹰，所有老师就给了他们一块手帕用来投票，觉得谁适合当老鹰就把票投个谁，并且投票具有传递性，比如A把票投给了B，B把票投给了C，而C就有了两票，不能自己投自己。最后要你求出能得到的最多的票是多少，拥有最多的票数的编号是哪些，编号从0开始。
这个题有点像前面我做的那个牛与牛的受欢迎的问题，其实意思就是问你到达点i的点数最多是多少，这样的点可能不止一个，那就都要输出来。还是用到强连通分量来做，先求出所有的强连通分量，因为一个强连通分量里的点是可以互相到达的，意思是这个强连通分量里的点的拥有的票数是一样的，都是n-1（n是分量里的点数），然后在缩点，缩点后会发现具有最多票数的是出度为0的点，但是要统计出度为0的点有多少个点可以到达这个点并不好做，所以就需要把图反一下，这样就是求入度为0的点最多可以到多少个点了，只要把入度为0的点dfs一遍就好了。
