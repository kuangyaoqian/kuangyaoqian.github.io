最短路练习题(一)

发表于 2017-07-25 | 分类于 acm ， 图论 ， 最短路

做了一周的最短路径的练习，很有感触，所以特地的写一波。

poj3268 Silver Cow Party(Dijkstra变形)

题意：（这题题意有点坑）编号为1-n的n个农场有n头牛要到标号为x的农场开party，这些牛很聪明，他们都会选择最短的路径去x农场，并且从x农场回来，有m条路可以走，这些路是单向的，所以去和回的路不一定相同，问你所有的牛来回所花的最短的时间里最长的是多少时间。

一开始我以为是要求第n头牛到第x农场来回所花的最短时间，所以果断的Dijkstra，结果wa。（看题很重要啊！）
然后我又开始作死的暴力Dijkstra算法，从1-n都用一遍Dijkstra算法，求出每头牛到x的最短时间，存起来，最后再加上从x回来的最短时间。结果TLE。
所以正确的方法是，把这个有向图的箭头的反过来。然后再来求x农场到各头牛的农场所花的最短时间。这个和每头牛到x的最短时间是一样的，只要一次Dijkstra算法就可以了。
最后再一次Dijkstra算法求回去的最短路径，最后把两次求出来的最短时间相加，求出这些时间里的最大值输出就ac 了！！

poj1797 Heavy Transportation(Dijkstra拓展)

大意很好理解：就是说要找出这个图中能承载的最大的重量，即要找到权值较大的路径中的最小的权值，然后输出。
原理和Dijkstra一样，就是利用dist的不断更新，但是条件和dist的定义要改一下，dist数组变为源点到i点的权值较大的路径上的最小的权值。注意最后一行还有一个换行。选点的时候选权值最大的点，但是用这个点向外更新的时候是选最小权值。（我把这种路径叫做最大路径最小值，即求的是最大权值的路径，但是是求这条路径上的最小权值）

hdu4725 The Shortest Path in Nya Graph（最短路spfa）

题意：有n个点，m条边，每条边都有一个权值，并且每个点属于一层，楼层上的所有点可以到相邻楼层的任意点，但是要花费c的代价，没有点的楼层不能互达，每组测试数据第一行是n，m，c，第二行是n个数 li表示第i个点属于的楼层。然后m行是三个数u，v，w表示m条边（无向的）。问从第一个点到第n个点的最小花费是多少。
这个题的关键是建图，把图建好了以后就是一个裸的最短路径，将楼层抽象出一个个点来（n+1~n+n），然后再点与点建边，双向的，权值为w；点与该点所在的层建边，权值为0，单向的；楼层与楼层建边，双向的，权值为c；点与相邻的楼层建边，单向的，权值为c。

lightoj1074 Extended Traffic(判断负环spfa)

题意：有n个城市，每一个城市有一个拥挤度ai，从一个城市I到另一个城市J的时间为:(aJ-aI)^3，存在负环。问从第一个城市到达第k个城市所花最短的时间，如果不能到达，或者时间小于3输出”?”，否则输出所花的最短时间。
Input
第一行输入一个整数T表示有T组测试数据，在一组测试数据的第一行输入一个
整数n，表示有n个城市，接着输入n个数表示1到n个城市的拥挤度，然后下一行输入一个整数m，表示有多少条边，接下来m行输入每行两个整数，表示从第i个城市可以到第j个城市，然后输入t组询问，每组表示一个要到达的城市。
Output
第一行输入”Case 1:”，然后接下来每一行一个整数表示从第一个城市到第k个城市所花的最短时间。

这题也简单只要spfa判断一下负环就好了

poj2253 Frogger(dijkstra拓展)

题意：有一只青蛙要到另一只青蛙那里去，给出n个点表示青蛙可以跳的点的坐标，要求求出所有路径中的最大权值中的最小值（真是变态的题意）。其实也就是说求较短路径中的最大权值，用Dijkstra算法写，每次选点的时候就选权值最小的边连着的点，用这个点向外更新的时候就选最大的更新。（我把这种路径叫做最小路径最大值，即求的是最小权值的路径，但是是求这条路径上的最大权值）
我是用邻接数组写的，所以M不能开的太大了。
